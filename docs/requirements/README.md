# Requirements

This folder holds curated product-intent briefs. Briefs are not executable specs; OpenSpec is the only executable-spec layer.

## Lifecycle

- candidate: extracted but not selected.
- ready-for-openspec: clear enough to become a change.
- in-openspec: linked to an active change.
- implemented: archived and reflected in accepted behavior.
- deferred: intentionally postponed.

## Required Brief Content

Use the template to record a stable ID, source, intent, actors, scope, out of scope, flows, constraints, dependencies, questions, and suggested change.

Update the brief and index coherently when a linked change archives.

Do not force a brief for a refactor, documentation update, or internal technical task that does not change an accepted contract. OpenSpec remains optional for such technical traceability.
